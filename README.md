# Make_own_ai_tic_tac_toe
Сойдай свою нейросеть для игры в крестики нолики 

1. Как устроена модель и код
Архитектура нейросети (_build_model)

    Вход (27 нейронов): Доска 3x3 (9 клеток). Каждая клетка может быть в 3 состояниях: крестик (1), нолик (-1), пусто (0). Код использует One-Hot Encoding (метод board_to_input), превращая 9 клеток в вектор из 27 элементов (9 клеток × 3 состояния).
    Скрытые слои: Три полносвязных слоя (512 -> 256 -> 128) с активацией ReLU, нормализацией (BatchNormalization) и регуляризацией (Dropout).
    Выход (9 нейронов): Для каждой из 9 клеток сеть выдает Q-значение — оценку того, насколько выгодно поставить фигуру в эту клетку.
    Две сети: В DQN используется основная сеть (для выбора действий и обучения) и целевая сеть (target_model), которая используется для расчета "правильных" ответов (targets). Это стабилизирует обучение.

Механика агента (DQNAgent)

    Replay Buffer (memory): Агент сохраняет свой опыт (состояние, действие, награда, следующее состояние) в очередь. При обучении он берет случайную выборку (minibatch), чтобы разорвать корреляцию между последовательными шагами.
    Epsilon-Greedy (act): С вероятностью epsilon агент делает случайный ход (исследование), иначе выбирает ход с максимальным Q-значением (использование опыта).
    Маскировка ходов: В методе act нелегальные ходы (занятые клетки) заменяются на -inf, чтобы нейросеть точно не выбрала их, даже если она ошибочно присвоила им высокий Q-вес.
